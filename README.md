# Peripheral Blood Cell Classification

MSt in Healthcare Data Science, Module 5 — University of Cambridge

## Overview

This project compares frozen backbone feature extraction against end-to-end fine-tuning for peripheral blood cell (PBC) classification across 13 cell types. Three pretrained backbones are evaluated — two ImageNet-pretrained CNNs (ResNet-50, EfficientNet-B0) and one domain-specific Vision Transformer (DinoBloom-S, pretrained on haematology images) — to assess whether domain-specific self-supervised pretraining can deliver clinically useful classification without GPU-dependent fine-tuning.

**Classifiers on frozen features:**
- Logistic regression (linear probe baseline)
- XGBoost with Optuna hyperparameter tuning (TPE sampler)

**End-to-end fine-tuning:**
- All three backbones fine-tuned with Optuna HP search, AdamW optimiser, cosine annealing LR scheduler, and Hyperband pruning

**Dataset:** [KU-Optofil](https://zenodo.org/records/14997735) — 31,484 single-cell images across 13 classes from 276 patients, with patient-level metadata enabling leakage-free train/validation/test splits.

## Setup

### Local (macOS / Linux)

```bash
conda env create -f environment.yml
conda activate classileukotion
```

For GPU support (CUDA), install PyTorch with the appropriate CUDA version:
```bash
pip install torch torchvision --index-url https://download.pytorch.org/whl/cu121
```

### Dependencies

If not using the conda environment, the key packages are:

- Python 3.11
- numpy, pandas, scikit-learn
- matplotlib, seaborn, pillow
- torch (>=2.1), torchvision
- xgboost (>=2.0)
- optuna (>=3.6)
- huggingface_hub
- umap-learn, shap, pytorch-grad-cam, timm

## Implementation

The various models are trained locally (on a M1 pro macbook) when possible, and offloaded to the CSD3 HPC when not. Use the setup scripts in the `hpc/` folder for setting up your hpc environment and sending jobs to a slurm-based scheduler. Reference resource usage is tabulated below.


| Job | Partition | Resources | Walltime |
|-----|-----------|-----------|----------|
| Feature extraction | icelake | 32 CPU, 64 GB | 2 h |
| XGBoost training | icelake | 32 CPU, 64 GB | 4 h |
| Fine-tuning | ampere (A100) | 1 GPU, 8 CPU, 32 GB | 4–6 h |

Per-backbone jobs can be submitted in parallel using `--backbone <name>` with `sbatch --wrap`. See `hpc/` scripts for details.
